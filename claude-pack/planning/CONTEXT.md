# Planning Context

Purpose:
Define what the game becomes over time without breaking the fun.

Read this file when deciding:
- what the next feature should be
- whether a feature is Layer 2 or Layer 3
- whether a change is worth the complexity

Planning principles:
- Keep the game first
- Make learning measurable
- Prefer systems over one-off features
- Only add features that create reusable value
- Avoid dependencies that can become user-facing blockers

The main product question:
Does this feature make the game smarter for the player, or just busier?

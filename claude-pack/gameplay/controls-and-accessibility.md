# Controls and Accessibility

Current strengths:
- desktop keyboard controls
- mobile control layer
- audio on/off controls
- voice on/off controls

Future requirements:
- make all major actions reachable on mobile
- keep the game playable without audio
- ensure color is not the only signal
- show clear pause, restart, and back-to-menu states

Accessibility rule:
No player should be blocked from learning because a single modality is missing.
